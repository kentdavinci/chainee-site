# Chainee网站

Last updated: 2026-05-18

## Project Identity

Project name: Chainee网站

Brand name: 链习社 Chainee

Goal: Design, build, and launch the official website for Chainee.

Current stage: Project setup, reference gathering, design direction, and implementation planning.

## Working Intent

The website should make Chainee feel credible, contemporary, and useful from the first screen. It should communicate what Chainee is, who it serves, why it matters, and how visitors can join, learn, collaborate, or follow future updates.

## Initial Website Objectives

- Establish a clear public identity for Chainee.
- Present the community, learning, and Web3/blockchain-related positioning in a polished way.
- Build a design system that can scale into future pages, events, articles, courses, or community products.
- Prepare the site for real deployment, not just a mockup.

## Skill Stack

Installed for this project:

- `figma-generate-design`: useful if we want to turn strategy and references into Figma-style page concepts.
- `figma-create-design-system-rules`: useful for turning visual decisions into reusable design-system guidance.
- `figma-implement-design`: useful if there is already a Figma design and we want to implement it faithfully.
- `screenshot`: useful for capturing and comparing reference sites or QA states.
- `playwright`: useful for testing the website across viewports and checking interactions.

Pending deployment choice:

- `vercel-deploy`: strong default for launching a modern frontend quickly.
- `netlify-deploy`: good alternative for simple static or Jamstack deployment.
- `cloudflare-deploy`: good if we want Cloudflare Pages, edge hosting, or Cloudflare-native infrastructure.

Already available in this Codex session:

- Browser automation for opening and verifying local pages.
- Image generation skill if we need bespoke visual assets.
- General frontend implementation support.

The reference research should focus on international websites with strong editorial hierarchy, premium product storytelling, and community credibility. Good categories to collect:

- Education and cohort-based learning platforms.
- Web3, crypto, and developer ecosystem sites.
- Modern community and creator network sites.
- Conference, event, and institute websites.
- Premium SaaS and foundation-style websites with strong information architecture.
