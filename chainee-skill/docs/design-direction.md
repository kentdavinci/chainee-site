# Chainee Visual Direction

Last updated: 2026-05-18

## Brand Name Treatment

Primary site name: 链习社 Chainee

Recommended usage:

- Navigation logo: 链习社 Chainee
- Hero title: 链习社 Chainee
- Supporting English line: Global access. Web3 tools. Practical routes.
- Chinese-first body copy, with selective English labels for structure and atmosphere.

The bilingual name should feel intentional rather than translated. Chinese gives the project warmth and community identity; English gives it international reach and a more contemporary product feel.

## What To Borrow From The References

### 1. Dark Technical Editorial System

The first and fourth references use black, graphite, thin lines, numerical panels, and abstract technical forms. This is a strong fit for Chainee because it suggests seriousness, systems thinking, and frontier technology.

Borrow:

- Black and deep gray backgrounds.
- Thin divider lines and dotted borders.
- Large compressed headings.
- Modular stat panels.
- Abstract mesh, chain, node, or orbital graphics.

Avoid:

- Overly industrial wording.
- Dense dashboards that make the site feel like oil, drilling, aerospace, or defense.
- Too much all-caps English, which can overpower the Chinese brand.

### 2. Light Futuristic Product Layout

The third reference has an airy off-white background, dotted typography, diagram lines, and central icon composition. This direction is useful for trust, readability, and education.

Borrow:

- Off-white sections to avoid making the whole site too dark.
- Central diagram or node-based hero composition.
- Dot-matrix details as a motif for knowledge, blocks, signals, and networks.
- Clean navigation and strong whitespace.

Avoid:

- Making Chainee look like a SaaS bug-fixing tool.
- Overusing dotted display typography where Chinese characters need legibility.

### 3. Premium Card Grid And Motion-like Imagery

The second reference uses rounded panels, image strips, stats, and high-contrast green-black 3D imagery. This can inform community programs, learning tracks, and outcomes.

Borrow:

- Alternating light and dark cards.
- Compact stat tiles.
- A restrained accent color.
- One strong cinematic visual band.

Avoid:

- Generic pricing-card structure unless Chainee has real paid programs.
- Too many rounded cards stacked inside each other.

### 4. Human + Future Imagery

The fifth reference introduces people, headsets, and future-facing portrait imagery. For Chainee this is helpful because the site should not feel like pure machinery.

Borrow:

- Human presence in selected sections.
- Dark immersive sections with large image moments.
- Soft contrast between futuristic tech and community warmth.

Avoid:

- VR-specific imagery unless Chainee actually wants that association.
- Stock-looking sci-fi faces that dilute authenticity.

## Recommended Chainee Direction

Chainee should sit between a practical overseas finance guide, a Web3 tool map, and a frontier technology community.

The strongest direction:

- Dark technical interface as the dominant first impression.
- Off-white sections only as contrast breaks, not as the main website tone.
- Black, graphite, and acid-green system visuals for depth and atmosphere.
- One accent color, likely electric green, signal orange, or cool cyan.
- Bilingual typography: clean Chinese sans-serif for credibility, technical English display type for small labels and section markers.
- Network, chain, node, signal, route, and archive motifs instead of generic crypto coins.

## Visual Reference Priority

The current homepage should prioritize the 1st, 3rd, 4th, and 5th reference images:

- 1st image: dark technical panels, metallic/system atmosphere, thin lines, instrument-like composition.
- 3rd image: dot-matrix, node-line diagrams, clean futuristic product structure.
- 4th image: oversized futuristic brand type, space-like depth, hard-edged technical sections.
- 5th image: black/white future-facing editorial rhythm, large typography, immersive dark sections.

The site should feel less like a standard SaaS landing page and more like a high-trust technical route console for global finance and Web3 access.

## Content Positioning

The site should not feel primarily like an education or course platform. Learning content exists, but only as one branch inside a broader practical route system.

The main content promise:

- Help users understand overseas finance access.
- Organize Hong Kong and US market investment entry points.
- Map crypto and Web3 tools.
- Explain account, banking, exchange, wallet, and fund-flow paths.
- Surface real-world limitations, risk notes, and community feedback.

Course-like content should be grouped under one branch, such as `Web3 学习分支`, rather than dominating the homepage.

## Typeface Direction

Chinese:

- Use a modern sans-serif with high legibility.
- Suggested system-first stack: PingFang SC, Noto Sans SC, Microsoft YaHei.
- Avoid decorative Chinese display fonts for body text.

English:

- Use a clean grotesk or geometric sans for body and navigation.
- Use a technical display style only for short labels, numbers, and hero accents.

Suggested pairing:

- Main UI/body: Inter or similar.
- Chinese: Noto Sans SC or PingFang SC.
- Technical accents: dot-matrix, mono, or condensed display treatment used sparingly.

## Homepage Structure Draft

1. Hero
   - Brand: 链习社 Chainee
   - One-line positioning.
   - Abstract node/chain visual or generated cinematic bitmap.
   - Primary action: 加入社区 / Join Chainee
   - Secondary action: 了解我们 / Explore

2. Manifesto
   - Explain why Chainee exists.
   - Use large editorial text, not a generic paragraph block.

3. Route System
   - Present overseas setup, banking, brokerage, crypto, wallet, fund flow, and risk-control steps.
   - Use modular route cards or technical panels.

4. Practical Library
   - Tools, guides, checklists, route maps, and risk notes.
   - Use stats carefully only if real numbers exist.

5. Community Proof
   - Feedback, route updates, real user questions, tool testing, salons, or project labs.

6. Join Section
   - Clear call to action.
   - WeChat, Discord, email, newsletter, or application form depending on available channels.

## Visual Motifs To Explore In Code

- Dot-grid brand mark derived from chain links or learning nodes.
- Thin-line network diagram around the hero.
- Abstract metallic or glass chain fragment as a generated image asset.
- Section labels like `01 / ORIGIN`, `02 / LEARN`, `03 / BUILD`, `04 / GATHER`.
- Stat panels, but only with truthful numbers.
- Dark technical footer with bilingual navigation.

## Design Guardrails

- Do not make the website look like a generic crypto token project.
- Do not use neon gradients as the main identity.
- Do not rely on coins, candlesticks, wallets, or speculative trading visuals.
- Keep Chinese text readable and dignified.
- Use futuristic effects as atmosphere, not as decoration overload.
- Make the first version feel real even if content is still evolving.
